
# Life Coach（对话记录助手）MVP阶段产品需求文档（PRD）
## 一、产品概述
### 1.1 产品定位
Life Coach 是一款 **AI驱动的对话记录与智能梳理工具**，聚焦个人/小团队场景，支持对话实时记录、语音转写、AI结构化整理、重点提取与复盘引导，核心解决手动记录低效、信息零散、难提炼价值的痛点，助力用户沉淀对话洞察、优化决策与复盘效率（适配口播创作、团队会议、个人思考对话等场景）。

### 1.2 核心目标（MVP阶段）
1.  实现对话记录（手动/语音）与本地存储核心功能，保障数据隐私可控；
2.  集成轻量AI能力，完成对话结构化整理、关键词提取与核心总结；
3.  提供基础检索与标签管理，方便用户快速定位对话内容；
4.  适配PC端核心场景，操作流程简洁，满足单人/小团队日常使用需求。

### 1.3 目标用户
1.  个人创作者（如AI口播博主）：记录脚本讨论、灵感对话，快速提炼口播要点；
2.  创业者/团队负责人：记录团队会议、合伙人沟通，沉淀决策要点与待办事项；
3.  自我成长者：记录个人思考对话、咨询沟通，通过AI引导完成复盘与反思。

### 1.4 核心价值
- 高效记录：语音转写（本地模型优先）+ 手动编辑，替代传统纸笔/纯文字录入；
- 智能梳理：AI自动结构化对话内容，提取重点、待办、疑问，减少整理成本；
- 隐私可控：本地存储为主，避免对话敏感信息（如商业想法、个人思考）上云泄露；
- 轻量化复用：支持标签、检索，方便对话内容二次复用（如口播素材、会议纪要）。

### 2.2 详细功能描述
#### 2.2.1 对话录入模块
1.  手动输入对话：
    - 支持区分“自己”“对方1”“对方2”角色（默认3个角色，可自定义角色名称）；
    - 输入框支持换行、分段，自动记录录入时间戳（精确到分钟）；
    - 支持实时保存，关闭页面后再次打开可恢复未完成对话。
2.  本地语音转写录入：
    - 支持麦克风实时收音转写，转写过程中显示实时文字，可暂停/继续；
    - 转写完成后自动区分发言停顿（按语音间隙分段），用户可手动调整角色归属；
    - 支持导入本地音频文件（MP3/WAV格式）转写，转写速度适配本地硬件性能；
    - 集成降噪功能（默认开启），优化环境噪音下的识别准确率。
3.  保存与备份：
    - 对话默认以“日期+对话主题（自动提取关键词）”命名，用户可手动修改；
    - 本地存储格式为加密文件（保障隐私），支持导出为TXT/MD格式；
    - 自动备份频率可选（每日/每周），备份文件存储于自定义路径下的“备份文件夹”。

#### 2.2.2 AI智能处理模块
1.  对话结构化整理：
    - 触发方式：手动点击“AI整理”按钮，或对话录入完成后自动触发（可在设置中关闭）；
    - 整理结果：按“对话主题-发言角色-核心观点-补充说明”结构呈现，支持折叠/展开各章节；
    - 适配场景：会议对话自动标注决策点、讨论点、争议点；口播对话自动标注素材要点、逻辑框架。
2.  关键词/要点提取：
    - 关键词：提取5-10个核心词汇（基于对话频率、重要性排序）；
    - 要点清单：以无序列表形式呈现，每条要点不超过20字，覆盖关键结论、核心需求、重要数据。
3.  总结与待办生成：
    - 总结：提炼对话核心内容，去除冗余话术，保留关键结论与行动方向；
    - 待办事项：自动识别对话中“需要做的事”，标注负责人（基于角色）、优先级（高/中/低），支持手动添加/删除/修改状态（未开始/进行中/已完成）。
4.  复盘引导提问（P1，可选功能）：
    - 基于对话类型（会议/创作/咨询）生成针对性问题，引导用户深化思考；
    - 问题示例：会议场景“本次会议确定的 deadlines 有哪些？是否存在风险？”；口播场景“本次灵感对话中，可转化为口播素材的点有几个？”。

#### 2.2.3 对话管理模块
1.  标签分类：
    - 支持为单条对话添加1-5个标签，可创建自定义标签（如“AI口播素材”“团队会议”“个人复盘”）；
    - 标签支持编辑、删除，可按标签筛选所有关联对话。
2.  检索功能：
    - 全局检索框支持输入关键词，检索结果按“匹配度”排序，高亮显示匹配内容；
    - 高级检索：支持组合筛选（标签+时间范围+角色），精准定位目标对话。
3.  基础编辑：
    - 可修改对话内容、调整角色归属、删除冗余段落；
    - 支持一键还原AI整理前的原始对话内容；
    - 编辑后自动更新修改时间戳，保留历史版本（MVP阶段保留最近1个历史版本）。

#### 2.2.4 基础设置模块
1.  用户信息管理：
    - 设置个人头像（本地上传）、昵称，作为对话中“自己”角色的默认标识；
    - 支持保存常用角色名称（如“合伙人A”“客户”），下次录入直接选择。
2.  存储配置：
    - 自定义对话存储路径（默认路径为“我的文档/Life Coach/对话记录”）；
    - 备份频率设置（每日/每周/关闭自动备份），备份文件命名格式为“原文件名_备份时间”。
3.  语音模型配置：
    - 模型选择：提供“轻量版”（优先速度，适合低配设备）、“标准版”（平衡速度与精度）两个选项；
    - 降噪设置：支持开启/关闭降噪功能，关闭后提升转写速度，适合安静环境。

## 三、交互设计需求
### 3.1 核心页面与流程
1.  首页（对话列表页）：
    - 布局：左侧标签栏（所有标签+未分类），中间对话列表（按时间倒序排列，显示标题、日期、标签、核心关键词），右侧快捷操作（新建对话、检索框）；
    - 交互：点击对话条目进入详情页，右键支持快捷操作（删除、导出、添加标签）。
2.  对话详情页（录入/编辑/AI处理页）：
    - 布局：顶部操作栏（保存、AI整理、导出、返回），左侧角色选择栏，中间内容区（原始对话/AI整理结果，可切换视图），右侧辅助区（关键词、待办、复盘问题）；
    - 交互：语音转写时显示“转写中”状态，转写完成后提示“可调整角色归属”；AI整理时显示加载动画，完成后自动切换至整理视图。
3.  设置页：
    - 布局：分“用户信息”“存储配置”“语音模型”“关于”四个标签页，每个标签页下按选项分组；
    - 交互：修改配置后实时保存，部分配置（如存储路径）修改后提示“需重启应用生效”。

### 3.2 关键交互规则
1.  角色切换：在对话录入时，点击左侧角色名称，即可切换当前发言角色，输入内容自动归属至该角色；
2.  AI处理触发：对话录入完成后（无操作30秒）自动触发AI整理（可关闭），也可手动点击“AI整理”按钮立即触发；
3.  待办状态修改：点击待办事项前的复选框，切换“未开始/已完成”状态，已完成项显示删除线；
4.  导出交互：支持导出原始对话、AI整理结果两种格式，导出时可选择是否包含关键词、待办事项。

## 四、非功能需求
### 4.1 性能需求
1.  启动速度：PC端启动时间≤3秒，无明显卡顿；
2.  语音转写速度：轻量模型下，实时转写延迟≤1秒/句话，本地音频转写速度≥1.5倍音频时长；
3.  AI处理速度：单条对话（≤1000字）AI整理时间≤5秒，复杂对话（1000-3000字）≤10秒；
4.  检索速度：关键词检索响应时间≤1秒，组合检索≤2秒。

### 4.2 隐私与安全需求
1.  数据存储：所有对话数据、音频文件默认本地存储，无强制上传云端功能，保障数据隐私；
2.  数据加密：本地存储文件采用AES加密，防止第三方访问；
3.  权限控制：仅本地当前用户可访问数据，无多用户共享功能（MVP阶段）。

### 4.3 兼容性需求
1.  系统兼容：支持Windows 10及以上、macOS 12及以上系统；
2.  硬件兼容：适配主流PC麦克风、耳机，最低配置要求：CPU i3及以上，内存8GB及以上，硬盘剩余空间≥10GB（用于存储模型与对话文件）。

### 4.4 稳定性需求
1.  语音转写过程中无崩溃、闪退现象，异常中断后可恢复已转写内容；
2.  AI处理过程中支持取消操作，取消后保留原始对话内容；
3.  每日自动备份功能稳定运行，无丢失对话数据风险。

## 五、上线标准（MVP阶段）
1.  功能完整性：所有P0级功能实现无缺失，核心流程（录入-AI处理-管理）可顺畅运行；
2.  性能达标：满足上述性能需求，在最低配置设备上无明显卡顿、延迟；
3.  兼容性：在主流PC系统（Windows 11、macOS Ventura）上测试通过，无兼容性BUG；
4.  易用性：核心操作路径≤3步，无复杂学习成本，新用户可快速上手；
5.  稳定性：连续72小时模拟日常使用场景，无崩溃、数据丢失问题，BUG率≤0.5%（关键功能无BUG）。

## 六、迭代规划（MVP后）
1.  V1.1版本：新增云端存储（可选，加密上传）、多端同步（PC+移动端）、对话分享功能；
2.  V1.2版本：优化AI能力，支持自定义结构化模板、多语言对话处理、语音情绪分析；
3.  V1.3版本：新增团队协作功能，支持多人共同编辑对话、角色权限管理、会议纪要模板库。
